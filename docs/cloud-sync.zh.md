# 离线版安全同步到云端

本功能适用于 Jeff 仍在办公室电脑使用离线版录入、同时希望手机通过云端查看真实订单的过渡阶段。

## 1. 推荐的数据主次关系

过渡期内保持：

- 办公室离线版是唯一录入端。
- 云端接收离线版推送，并供手机或其他电脑查询。
- 云端设置 `JEFF_CLOUD_SYNC_READ_ONLY=true`，普通登记、修改、交货和核销操作会被服务器拒绝。

正式切换当天完成最后一次同步和抽查后，再把云端改为唯一录入端，并将上述变量改为 `false`。不建议长期把本地和云端同时当成两个独立主库。

## 2. 首次配对

1. 管理员登录云端，打开“数据工具 → 离线版同步管理”。
2. 点击“生成一次性配对码”。配对码十五分钟内有效，成功使用一次后立即失效。
3. 在 Jeff 办公室电脑打开“云端同步”，填写页面显示的 HTTPS 服务器地址和六位配对码。
4. 配对成功后，离线版保存一个仅具备同步上传权限的设备令牌，不保存管理员密码。

Windows 离线版使用当前 Windows 用户的 DPAPI 加密令牌。加密结果保存在 `data/cloud-sync-client.json`，原始令牌不会写入文件。复制这个文件到另一台电脑也不能解密使用。

云端把设备令牌的哈希、设备名称和同步记录保存在 `data/cloud-sync-devices.json`。管理员可以随时撤销某台电脑的同步授权。

## 3. 日常同步

配对后默认开启自动同步：

1. 本地订单发生变化后等待约十五秒，合并短时间内的连续操作。
2. 软件通过 SQLite Backup API 生成一致性快照，自动包含 WAL 中的数据。
3. 快照通过 HTTPS 上传，并携带本机数据版本、软件版本和数据库结构版本。
4. 云端检查文件大小、SQLite 完整性、结构和 SHA-256，再调用现有三方迁移引擎。
5. 没有冲突时自动创建云端迁移前备份，并在事务中完成安全合并。
6. 网络中断或服务器暂不可用时，本地数据库不受影响，软件稍后自动重试。

Jeff 也可以随时点击“立即同步”。界面只显示以下主要状态：

- 数据已同步。
- 本地有新修改。
- 等待管理员处理冲突。
- 暂时无法同步，本地数据仍安全。

## 4. 合并和冲突

自动规则与手动云端迁移一致：

| 差异 | 处理方式 |
| --- | --- |
| 仅本地新增 | 新增到云端 |
| 基线后仅本地修改 | 采用本地 |
| 仅云端存在或仅云端修改 | 保留云端 |
| 两边相同 | 不重复写入 |
| 同一订单两边都修改 | 暂停并等待管理员逐单选择 |

发现冲突时不会自动覆盖。管理员在“离线版同步管理”点击“逐单处理”，进入现有 `/migration` 页面选择“采用本地”或“保留云端”。完成后 Jeff 的离线版会在下次状态检查时显示同步完成。

每次实际合并前都会生成一致性备份，使用维护锁和数据库事务，失败时自动恢复。重复上传同一备份会被识别，不会生成重复订单。

## 5. 地址要求

互联网同步必须使用固定 HTTPS 地址。临时 `trycloudflare.com` Quick Tunnel 地址在隧道重启后可能变化，只适合短期验证。

正式试用推荐使用 Cloudflare Named Tunnel 和固定域名，即使 Windows 云桌面没有公网 IP 也可以工作。私有局域网地址可以使用 HTTP；公网 HTTP 默认会被离线版拒绝。

## 6. 验证命令

```bash
npm run test:cloud-migration
npm run test:cloud-sync
npm run test:local-sync
npm run lint
npm run build
```

`test:local-sync` 在 Windows 上实际验证 DPAPI 加密、令牌不明文落盘、一致性备份上传和授权请求；非 Windows 环境会跳过 DPAPI 测试。
